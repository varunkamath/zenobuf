1//! # Zenobuf Examples - Example applications for the Zenobuf framework
2//!
3//! This crate contains comprehensive examples demonstrating how to use the Zenobuf
4//! framework for building distributed systems. Each example showcases different
5//! aspects of the framework and provides practical, runnable code.
6//!
7//! ## Available Examples
8//!
9//! ### Basic Communication
10//!
11//! - **[talker.rs](src/bin/talker.rs)** - Publisher example that sends geometry messages
12//! - **[listener.rs](src/bin/listener.rs)** - Subscriber example that receives geometry messages
13//!
14//! ### Service Communication
15//!
16//! - **[service.rs](src/bin/service.rs)** - Service example that provides addition functionality
17//! - **[client.rs](src/bin/client.rs)** - Client example that calls the addition service
18//!
19//! ### Advanced Examples
20//!
21//! - **[complete_app.rs](src/bin/complete_app.rs)** - Complete application with pub/sub, services, and parameters
22//! - **[parameters.rs](src/bin/parameters.rs)** - Parameter system usage example
23//!
24//! ## Running the Examples
25//!
26//! ### Publisher/Subscriber Example
27//!
28//! ```bash
29//! # Terminal 1: Start the listener
30//! cargo run --bin listener
31//!
32//! # Terminal 2: Start the talker
33//! cargo run --bin talker
34//! ```
35//!
36//! ### Service/Client Example
37//!
38//! ```bash
39//! # Terminal 1: Start the service
40//! cargo run --bin service
41//!
42//! # Terminal 2: Call the service
43//! cargo run --bin client 5 3
44//! ```
45//!
46//! ### Complete Application
47//!
48//! ```bash
49//! # Run the complete example (includes everything)
50//! cargo run --bin complete_app
51//! ```
52//!
53//! ### Parameter Example
54//!
55//! ```bash
56//! # Run the parameter example
57//! cargo run --bin parameters
58//! ```

60//! ## Message Types
61//!
62//! The examples use Protocol Buffer messages defined in the `protos/` directory:
63//!
64//! ### Geometry Messages (`protos/geometry.proto`)
65//!
66//! - **Point** - 3D point with x, y, z coordinates
67//! - **Quaternion** - Orientation representation
68//! - **Pose** - Position and orientation combined
69//!
70//! ### Service Messages (`protos/example_service.proto`)
71//!
72//! - **AddTwoIntsRequest** - Request with two integers to add
73//! - **AddTwoIntsResponse** - Response with the sum
74//!
75//! ## Key Concepts Demonstrated
76//!
77//! ### 1. Node Creation
78//!
79//! ```rust,no_run
80//! use zenobuf_core::Node;
81//!
82//! #[tokio::main]
83//! async fn main() -> Result<(), Box<dyn std::error::Error>> {
84//!     let node = Node::new("my_node").await?;
85//!     // Use the node...
86//!     Ok(())
87//! }
88//! ```
89//!
90//! ### 2. Publishing Messages
91//!
92//! ```rust,no_run
93//! # use zenobuf_core::Node;
94//! # use zenobuf_examples::proto::geometry::Point;
95//! # async fn example() -> Result<(), Box<dyn std::error::Error>> {
96//! # let node = Node::new("publisher").await?;
97//! let publisher = node
98//!     .publisher::<Point>("points")
99//!     .build()
100//!     .await?;
101//!
102//! let point = Point { x: 1.0, y: 2.0, z: 3.0 };
103//! publisher.publish(&point)?;
104//! # Ok(())
105//! # }
106//! ```
107//!
108//! ### 3. Subscribing to Messages
109//!
110//! ```rust,no_run
111//! # use zenobuf_core::Node;
112//! # use zenobuf_examples::proto::geometry::Point;
113//! # async fn example() -> Result<(), Box<dyn std::error::Error>> {
114//! # let node = Node::new("subscriber").await?;
115//! let _subscriber = node
116//!     .subscriber::<Point>("points")
117//!     .build(|point| {
118//!         println!("Received: ({}, {}, {})", point.x, point.y, point.z);
119//!     })
120//!     .await?;
121//!
122//! node.spin().await?;
123//! # Ok(())
124//! # }
125//! ```
126//!
127//! ### 4. Creating Services
128//!
129//! ```rust,no_run
130//! # use zenobuf_core::{Node, Result};
131//! # use zenobuf_examples::proto::service::{AddTwoIntsRequest, AddTwoIntsResponse};
132//! # async fn example() -> Result<()> {
133//! # let node = Node::new("service").await?;
134//! let _service = node
135//!     .service::<AddTwoIntsRequest, AddTwoIntsResponse>("add")
136//!     .build(|request| {
137//!         Ok(AddTwoIntsResponse {
138//!             sum: request.a + request.b,
139//!         })
140//!     })
141//!     .await?;
142//!
143//! node.spin().await?;
144//! # Ok(())
145//! # }
146//! ```
147//!
148//! ### 5. Calling Services
149//!
150//! ```rust,no_run
151//! # use zenobuf_core::{Node, Result};
152//! # use zenobuf_examples::proto::service::{AddTwoIntsRequest, AddTwoIntsResponse};
153//! # async fn example() -> Result<()> {
154//! # let node = Node::new("client").await?;
155//! let client = node
156//!     .client::<AddTwoIntsRequest, AddTwoIntsResponse>("add")
157//!     .build()?;
158//!
159//! let request = AddTwoIntsRequest { a: 5, b: 3 };
160//! let response = client.call(&request)?;
161//! println!("Result: {}", response.sum);
162//! # Ok(())
163//! # }
164//! ```

166//! ## Learning Path
167//!
168//! 1. **Start with Basic Examples**: Run `talker` and `listener` to understand pub/sub
169//! 2. **Try Service Communication**: Run `service` and `client` to understand RPC
170//! 3. **Explore Parameters**: Run `parameters` to see configuration management
171//! 4. **Study Complete Application**: Run `complete_app` to see everything together
172//! 5. **Build Your Own**: Use these examples as templates for your applications
173//!
174//! ## Tips for Development
175//!
176//! - **Use the CLI tools**: `zenobuf-cli list topics` to see active topics
177//! - **Monitor messages**: `zenobuf-cli monitor <topic>` to debug message flow
178//! - **Check services**: `zenobuf-cli list services` to see available services
179//! - **Manage parameters**: `zenobuf-cli param get/set` for runtime configuration
